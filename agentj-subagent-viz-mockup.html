Now — ASCII rail the problem

Box-drawing chars don't align in a proportional font; status/timing/tokens are cramped into one dim line.
├─┬─ ● Architecture options · bash({"command":"pwd && git status…"}) — 1.1k tok │ ╰─ ● Repo survey · bash({"command":"git status --branch…"}) — 1.5k tok ├─╯ wave 1 · 0/2 · 0ms · 2.6k tok

Option A — Task rows custom, inline

A small group box: a header, one scannable row per subagent (type chip · title · live status · tokens · elapsed), and a join footer. An indeterminate bar marks running rows. Best fit for an inline transcript.
Library: none needed — a purpose-built component. (This is what most agent UIs do.)

Option D — Rows + duration bars recommended

Option A, plus each row carries a duration bar so you SEE the parallelism and where wall-clock goes (the wave is as slow as its slowest agent). Running bars animate; finished bars show their span. Scannable and temporal, still fits inline.
Library: none needed (thin CSS bars). If you later want a full gantt: visx or nivo.

Option C — Timeline lanes temporal

A mini time axis with one lane per agent — bars start together and end at each finish. Makes overlap and the wall-clock obvious, but takes more vertical space and reads less like a task list.
0s10s20s25s
Library: visx / nivo for a real gantt, or custom CSS (shown).

Option B — Fork/join graph DAG

An actual node graph: the delegate node fans to N task nodes and joins. Great for a dedicated, zoomable panel (like the old epic DAG), but an interactive canvas inside a scrolling log is awkward and heavy.
Library: React Flow (@xyflow/react) — the standard for node/DAG graphs (auto-layout via dagre/elkjs).
My take: Option D (rows + duration bars) for the inline transcript — it fixes the alignment/scan problems and adds the one genuinely useful extra (where the parallel wall-clock goes), with zero heavy deps.
