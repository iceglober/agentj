agentj · TUI redesign

A live view that shows state, not just stream

Today every surface — the manager's turn, each engineer, subagent waves, background jobs — interleaves into one scrolling transcript plus side panels. This keeps the conversation primary, pins work state above it (as much or as little as the mode needs), and folds the low-level machinery into summaries you can open.

Anatomy — three zones, one adapts

Every screen is the same three stacked zones. Two are constant; the middle one changes weight with what you're doing.

status barAmbient, one line, always: model · context meter · tokens · elapsed.
work panelAdapts to the mode. Epic → the full ledger. A task in flight → a SPEAR spine + live frontier. Just chatting → it's gone. It summarizes the machinery's state, never the dialogue.
conversationConstant and primary. Your prompts and the assistant's answers as first-class blocks you read. Tool calls, notes, and subagent rails fold into one-line summaries so the answer stays clean.
composerConstant, pinned at the bottom — where prompts are sent. Enter sends, Ctrl-J adds a line, / opens commands. A sent prompt echoes as the next user block and the turn begins.

Principles

  • State over stream. The ledger and the SPEAR spine answer "where are we" without reading scrollback. The transcript stays, demoted to a collapsed detail strip.
  • Encode state in form. Status chips, a progress meter, step-budget and check bars — attention lands by color and shape before you read a word.
  • Unify the three activity surfaces. Jobs panel + subagent tray + transcript spam collapse into one thing: rows in the work panel, each with live progress.
  • Calm hierarchy. Status bar (ambient) → work panel (the answer) → activity spine (now) → transcript (detail). Fewer things move on screen.
  • Long work looks normal. An engineer at step 44/80 reads as productive, and a checkpointed one is a distinct chip — matching the step-limit resilience already in the loop.

What it takes to build

  • New src/tui/view/work.rs — renders the work panel from App state (ledger table / SPEAR spine + frontier), sitting between the status line and the transcript in view/mod.rs's layout.
  • A light live model on App — reuse what's cheap: engineer rows come from the existing jobs snapshot (they're jobs — elapsed/tokens already tracked); the ledger table reads .aj/epic/plan.md; the SPEAR phase and frontier items derive from supervisor notes + the frontier file. Richer per-PR check/thread state is a follow-up that reads gh.
  • One structured event (optional, phase 2) — an AgentEvent::Phase / LedgerTick so the panel updates crisply instead of re-deriving; not required for v1.
  • Keys — Tab focuses/expands a work-panel row; a Ctrl-P toggle shows/hides the panel. Reuses the existing menu + keymap plumbing.
  • Reuse — human_tokens, fmt_ms, the status line, and theme are already there; this is composition, not new rendering primitives.

Honest scope: v1 renders what's cheaply available (engineer rows from jobs, the ledger file as the table, SPEAR from notes). Per-PR check state and review-thread counts want live gh queries and are a phase-2 enrichment — the layout already has the column for them.
